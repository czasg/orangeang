静态文件服务器

Nginx 搭建静态文件服务器，通常使用root或者alias指令。

此外还需要指定一个目录作为静态文件存储路径配置。我们先假设有以下目录结构：

/static
  ├── css
  │   └── custom.css
  ├── js
  │   ├── main.js
  │   └── custom.js
  ├── unknown.html
  ├── notfound.html
  └── index.html

在这个目录结构中，我们可以看到根目录/static下面存放了html、js、css等文件。为了后续方便理解，我们假设每个文件的内容就是文件自身的详细路径~（即文件custom.css内容为/static/css/custom.css）

这样我们就可以开始准备 nginx 配置文件了。

root​

在 nginx 中，root关键字可以指定一个目录作为静态文件存储路径配置。

当请求命中路由规则时，会将路径拼接在root路径之后去查找文件！

其语法结构为：

root   dir;

为了更好的理解root的作用，我们假设定义了如下的路由规则：

location / {
    root   /static;
}

当请求路径为/static/css/custom.css时，我们会发现 nginx 查找文件的路径为/static/static/css/custom.css！

因此该请求无法找到对应的文件！

所以我们需要调整请求路径为/css/custom.css，这时 nginx 就可以正确命中/static/css/custom.css文件。

大致了解root的原理之后，我们可以开始准备 nginx 配置文件。

1. 准备配置文件​

/conf/root.conf
server {
    listen       80;
    server_name  _;

    # 静态文件服务器
    location /static {
        # 指定根目录
        root   /;
        # 指定默认返回的文件
        index  index.html;
    }
    # 文件不存在
    error_page  404  /static/notfound.html;
}

在上述配置中，我们定义了一个location /static规则，它表示所有以/static开头的请求都会命中此规则。并且规则中指定了根目录为/，这样我们就可以将此目录映射到静态文件服务器中了。

补充

error_page 可以检测响应的错误码，从而实现内部转发能力。

2. 启动服务​

$ docker run -it --rm -p 80:80 \
    -v /conf/root.conf:/etc/nginx/conf.d/default.conf \
    -v /static:/static \
    nginx

3. 验证功能​

通过浏览器访问http://localhost/static/ ，如果可以看到index.html页面，那就说明规则已经生效。

http://localhost/static/

/static/index.html

通过浏览器访问http://localhost/static/css/custom.css ，如果可以看到custom.css页面，那就说明规则已经生效。

http://localhost/static/css/custom.css

/static/css/custom.css

通过浏览器访问http://localhost/static/123 ，这是一个不存在的页面，这时我们如果可以看到notfound.html页面，那就说明规则error_page已经生效。

http://localhost/static/123

/static/notfound.html

此时notfound.html页面的查找逻辑为：

  • 请求/static/123返回404不存在
  • 规则error_page探测到404从而转发到/static/notfound.html
  • 请求/static/notfound.html并返回页面

此时用户看到的就是notfound.html页面，且状态码仍旧是404~~

alias​

在 nginx 中，alias和root一样可以指定服务器上静态文件的根目录。

但他们之间有一些细微的差别，前者不会拼接请求路径，而后者会拼接请求路径！

其语法结构同样为：

alias   dir;

为了更好的理解alias的作用，我们假设有如下的路由规则：

location / {
    alias   /static;
}

当请求路径为/static/css/custom.css时，我们会发现 nginx 查找文件的路径为/static/css/custom.css！

这正好可以找到目标文件！

而同样的请求在root指令下则会命中/static/static/css/custom.css!

大致了解alias和root区别之后，我们可以开始准备 nginx 配置文件。

1. 准备配置文件​

/conf/alias.conf
server {
    listen       80;
    server_name  _;

    # 静态文件服务器
    location /static {
        # 指定根目录
        alias   /static;
        # 指定默认返回的文件
        index  index.html;
    }
    # 文件不存在
    error_page  404  /static/notfound.html;
}

在上述配置中，我们定义了一个location /static规则，它表示所有以/static开头的请求都会命中此规则。并且规则中指定了根目录为/static，这样我们就可以将此目录映射到静态文件服务器中了。

2. 启动服务​

$ docker run -it --rm -p 80:80 \
    -v /conf/alias.conf:/etc/nginx/conf.d/default.conf \
    -v /static:/static \
    nginx

3. 验证功能​

通过浏览器访问http://localhost/static/ ，如果可以看到index.html页面，那就说明规则已经生效。

http://localhost/static/

/static/index.html

通过浏览器访问http://localhost/static/css/custom.css ，如果可以看到custom.css页面，那就说明规则已经生效。

http://localhost/static/css/custom.css

/static/css/custom.css

提示

细心的同学可能已经发现root和alias示例中只有根目录配置不同~😁😁😁

autoindex​

在 Nginx 中，autoindex指令用于启动或禁止目录列表功能。

也就是说，当我们启动了autoindex之后，我们就可以拥有一个在线文件浏览功能。这在调试和开发过程中非常有用，也可以用于提供文件下载服务。

通常来讲，autoindex指令可以在 http、server、location 块中使用。其语法结构为：

autoindex   on | off;

下面我们通过一个简单的示例来快速熟悉autoindex功能~

1.准备配置文件​

/conf/autoindex.conf
server {
    listen 80;
    server_name _;

    location / {
        root      /static/;
        autoindex on;
        # 不指定默认返回文件
        index     _;
    }
}

在上述配置中，我们在location中指定了autoindex on;，表示启用目录列表功能。接着我们启动服务。

2.启动服务​

$ docker run -it --rm -p 80:80 \
    -v /conf/autoindex.conf:/etc/nginx/conf.d/default.conf \
    -v /static:/static \
    nginx

3.验证功能​

通过浏览器访问http://localhost/ ，如果可以看到如下类似的列表页面，那就说明规则已经生效。
