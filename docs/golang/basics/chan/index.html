通道

在 Go 中，通道chan是一种类似队列的数据结构，可以用于不同 Go 协程之间传递数据。

通道可以分为无缓冲通道和有缓冲通道。

对于无缓冲通道来说，它需要通道两端同步读取和推送，才能进行通信。如果只有一端读取或者推送，那么将会发生阻塞直至另一端有协程同步操作。 如果你以前有过类似queue.get(block=True)/queue.put(block=True)的使用经验，那完全可以把二者等效看待。

对于有缓存通道来说，它有一定的缓冲区，因此它支持一定程度上的单向操作。比如单向推送场景中，通道先将数据放到缓冲区，因此不需要对端同步操作就可以完成推送操作。 但是当缓冲区满了的时候，有缓冲通道表现的就像无缓冲通道一样了。

初始化​

通道的语法结构为：chan 数据类型

不同于切片和映射，chan只能使用make进行初始化。并可以通过make指定缓冲区的大小。

无缓冲通道
ch := make(chan int)
有缓冲通道
ch := make(chan int, 10)

操作通道​

在 Go 中，通道的推送/读取操作都是通过左箭头<-实现的。

  1. 推送：语法结构为：chan <- 数据
有缓冲通道
ch := make(chan int, 10)
ch <- 1 // 非阻塞
ch <- 1 // 非阻塞
ch <- 1 // 非阻塞
  1. 读取：语法结构为：数据 := <- chan
有缓冲通道
ch := make(chan int, 10)
for i := 0; i < 10; i++ {
    ch <- i
}

v1 := <- ch
v2 := <- ch
v3 := <- ch
fmt.Println(v1, v2, v3)

关闭通道​

在 Go 中，可以使用内置函数close关闭一个通道。同时需要注意，已关闭的通道在 go 中会有一些特殊表象，因此对于已经关闭的通道需要谨慎处理。

  1. 对于一个已经关闭的通道进行读操作

如果是有缓冲且仍然有值未取出，这时通道是可以正常返回值的。但是一旦值全部取出，我们知道通道就像无缓冲通道一样，这时通道仍然可以返回值，不过返回的是指定数据类型的零值。

为了更好的区分通道是关闭还是未关闭，在读操作时支持第二个布尔类型的参数，用于判定当前通道是否处于关闭状态。

package main

import "fmt"

func main() {
	ch := make(chan int, 10)
	for i := 0; i < 3; i++ {
		ch <- i
	}

	close(ch) // 关闭通道

	for i := 0; i < 6; i++ {
		v, ok := <-ch
		fmt.Println(v, ok) // 即使通道已经关闭，仍然可以读取数据
	}
}

1 true 通道未关闭
2 true 通道未关闭
0 false 通道已关闭
0 false 通道已关闭
0 false 通道已关闭
