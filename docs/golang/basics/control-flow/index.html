流程控制

编程是一项极具逻辑思维的工作，流程控制则是编程的基础。

在编程过程中，开发者通过编写特定的代码实现流程控制，比如条件判断、循环处理以及中断等，通过这些方法开发者能够实现更为复杂的业务逻辑和功能。

条件语句（if）​

在 Go 中，if-else是常用的条件判断语句，其基本结构如下：

if 条件 {
  // code
} else if 条件 {
  // code
} else {
  // code
}

下面通过一些简单的示例来演示下条件语句的使用：

op := "orange"

if op == "coding" {
  // code
  fmt.Println("it's coding")

} else if op == "bug" {
  // code
  fmt.Println("it's bug")

} else {
  // code
  fmt.Println("unknown op")
}

在上述示例中，我们可以看到不同的op值会触发不同的代码块，整体逻辑相对清晰。当然，如果你运气足够好，在工作中遇到过if地狱，那么你对下面代码可能会感到额外亲切~

if地狱
if number >= 0 {
    if number >= 10 {
        if number >= 20 {
            if number >= 30 {
                if number >= 40 {
                    if number >= 50 {
                        if number >= 60 {
                            if number >= 70 {
                                if number >= 80 {
                                    if number >= 90 {
                                        fmt.Println("it's nb plus")
                                    }
                                }
                            }
                        }
                    }
                }
            }
        }
    }
}

循环语句（for）​

在 Go 中，for是常用的循环语句，其基本结构如下：

for 变量声明; 条件判断; 变量迭代 {
  // code
}

我们可以看到一个循环由三部分组成：

  • 变量声明：变量初始化，比如count := 0
  • 条件判断：条件判断逻辑，比如count <= 100
  • 变量迭代：变量迭代逻辑，比如count++

但实际上循环语句并不强制要求这三个模块，因此一个简单的for也可以有很多不同的玩法~ 下面演示一些常见的使用方法：

http://localhost:3000
  • 普通循环
  • 无限循环
  • 自定义变量迭代
for count := 0; count <= 100; count++ {
    fmt.Println("这是一个普通循环")
}
for {
    fmt.Println("这是一个无限循环")
}
for count := 0; count < 100; {
    // 把变量迭代放到代码块中，以实现更复杂的变量迭代
    if count < 50 {
        // 指数增长
        count *= 2
    } else {
        // 线性增长
        count++
    }
}

在上述示例中我们可以看到，每一个循环都可以通过设置条件判断来限制循环次数，如果没有条件判断的限制，那么这将是一个无限循环。

当然如果只靠循环本身，能实现的功能是有限的。因此，在循环中，还可以结合跳转语句来实现更多样的逻辑控制：

  • 跳转语句-break：提前结束一个循环
for count := 0; count <= 100; count++ {
  if count == 50 {
    fmt.Println("提前跳出循环")
    break
  }
}
  • 跳转语句-continue：提前进入下一个循环
for count := 0; count <= 100; count++ {
  if count < 50 {
    fmt.Println("提前进入下一次循环")
    continue // 提前进入下一次循环，不会执行后面的代码
  }
  break
}

选择语句（switch）​

在 Go 中，switch是常用的选择语句，通常情况下和if-else是等价的。其基本结构如下：

switch value {
case value1: // equal `if value == value1`
    // code
case value2: // equal `if value == value2`
    // code
default:     // equal `else`
    // code
}

在上述语法结构中，我们可以等价的把每一个 case 项比作一个 if 条件判断，因此不少开发者认为使用 switch-case 通常比 if-else 更加优雅。

需要注意的是，switch 支持 fallthrough 关键字，使用后会执行下一个 case 分支，这通常会破坏 case 分支的独立性，影响程序可阅读性，因此大部分场景下是不建议使用的。

http://localhost:3000
  • 常规使用
  • fallthrough
package main

import (
    "fmt"
)

func main() {
    value := "value"

    switch value {
    case "分支1":
        // code
        fmt.Println("命中分支1")

    case "分支2":
        // code
        fmt.Println("命中分支2")

    default:
        fmt.Println("default")
    }
}
package main

import (
    "fmt"
)

func main() {
    value := "value"

    switch value {
    case "分支1":
        // code
        fmt.Println("命中分支1")
        fallthrough // 会执行分支2的代码块

    case "分支2":
        // code
        fmt.Println("命中分支2")

    default:
        fmt.Println("default")
    }
}

跳转语句（goto）​

在 Go 中，goto是可选的跳转语句之一，它的作用是引导代码逻辑跳转到指定的锚点位置。在某些特殊场景下会让代码非常简洁，但滥用goto导致的屎山代码也是很常见的。

使用goto之前需要先定义一个锚点，定义规范为：锚点 + 冒号。下面通过一个简单的示例演示下goto的使用方法：

package main

import "fmt"

func main() {
    status := "success"

    switch status {
    case "success":
        goto SUCCESS
    case "failure":
        goto FAILURE
    default:
        return
    }

SUCCESS:
    fmt.Println("success")
    return
FAILURE:
    fmt.Println("failure")
    return
}

在上述代码中，我们定义了SUCCESS和FAILURE两个锚点，然后在代码逻辑中，我们可以通过goto关键字直接跳转到指定的锚点，从而简化程序执行流程。

延迟语句（defer）​

延迟语句是 Go 语言中非常具有特色的设计之一。

在 Go 中，开发者可以通过defer来实现延迟函数的的调用。由于它具有在程序panic之后仍然可以正常执行的特性，因此defer经常被用于资源的释放，比如文件句柄，锁等资源对象。

在使用过程中需要注意以下几点：

  • 后进先出：在函数中，可以定义多个延迟函数，他们遵循后进先出的原则，即最后定义的最先执行。
  • return后执行：在函数中，延迟函数定义后不会立即执行，而是等待业务代码执行完并return后，才会开始执行。
http://localhost:3000
  • 文件操作
  • 锁操作
package main

import "os"

func main() {
    f, err := os.Open("filename")
    if err != nil {
        panic(err)
    }
    defer f.Close()

    // code
}
package main

import "sync"

var lock = sync.Mutex{}

func main() {
    lock.Lock()
    defer lock.Unlock()

    // code
}
